## 4. 전개 구조 (5단계 공식)

```
[1] 인사 + 주제 소개 (2~4줄)
     ↓
[2] 고객 페인포인트 또는 트렌드 언급 (문제 제기, 공감 유도)
     ↓
[3] 본론: 소제목 + 설명 + 시각 요소 설명 (이미지 캡션 형태)
     ↓
[4] 마무리 + CTA (문의 요청)
     ↓
[5] 해시태그 (15~20개)
```

## 5. 말투 & 문체 규칙

### 기본 문체
- **경어체** 사용 (`합니다 / 습니다 / 니다`) — Key Copy(소제목)는 명사구/어절 단위, 설명 문단은 경어체로 구분
- 친근하되 전문적인 톤
- 독자에게 직접 말 걸기: `"~하고 계신가요?"`, `"~하셨나요?"`
- **본문은 짧고 담백하게.** 이미지가 주인공이므로 설명 과다 금지. 분량 기준은 섹션 7 참고

### 이모지 활용
- 알지비뉴스 카테고리: 이모지 적극 활용 (`⭐`, `🔥`, `✅`, `→`, `?`)
- 케이스스터디/홈페이지: 이모지 거의 없음, 깔끔한 텍스트 중심
- 정부지원사업: 이모지 최소화, 공식 문서 스타일 + 말미에 이모지 1~2개

### 네이버 SEO 키워드 반복 기법
핵심 키워드를 **굵게** 처리 후 바로 다음 줄에 한 번 더 반복 (네이버 검색 최적화).

예시:
```
기업 홈페이지 제작 및 UIUX 설계의 중요성
기업 홈페이지 제작 및 UIUX 설계의 중요성도 함께 커지고 있습니다.
```
```
3D 기반 웹디자인과 인터랙션 UI/UX 설계
3D 기반 웹디자인과 인터랙션 UI/UX 설계를 중심으로 작업을 진행했습니다.
```

### 불릿 포인트 활용
3개 전후의 불릿으로 핵심 정리:
```
- 복잡한 구조를 시각적으로 표현
- 보이지 않는 개념을 직관적으로 전달
- 핵심 메시지를 짧은 시간 안에 전달
```

### 소제목 스타일
```
고객의 이해를 돕는
3D 모델링
```
또는
```
기업 이미지를 가장 먼저 보여주는 Key Visual
```
(소제목은 짧게, 이어지는 설명은 첫 문장에서 소제목 키워드 반복)

## 13. 작업물 저장 경로 규칙

- 기본 저장 경로: `blog/번호.[카테고리]브랜드명/` 폴더 하위
- 초안(MD)과 최종본(HTML) 같은 폴더에 나란히 저장
- 예시:
  - `blog/08.[홈페이지]골프존클라우드/blog_post.md` (초안)
  - `blog/08.[홈페이지]골프존클라우드/blog_post.html` (최종본)
- 번호는 기존 폴더 순서에 맞춰 부여

### 작업 순서
1. MD 초안 작성 → 검토·수정
2. 확정 후 HTML 최종본 변환 저장

## 15. HTML 출력 규격

네이버 블로그 에디터 붙여넣기 최적화를 위해 `.html` 파일로 산출한다.
Chrome에서 열어 Ctrl+A → Ctrl+C → 네이버 에디터 Ctrl+V 방식으로 사용.

### 기본 구조 규칙

| 요소 | HTML 처리 |
|------|-----------|
| 전체 정렬 | `text-align: center` |
| 볼드 키워드 | `<strong>` |
| 섹션 제목 (Key Copy) | `<blockquote class="title">` — 상하 보더, bold 20px |
| 고객 인용문 | `<blockquote class="quote">` — 좌측 보더, italic 24px |
| 이미지 삽입 위치 | `<div class="img-placeholder">` — 파란 점선 박스 |
| 줄바꿈 | 모바일 기준 20자 내외로 `<br>` 수동 삽입 |
| 해시태그 | `.hashtags` 클래스, 14px |

### 이미지 프롬프트 포함 방법
이미지 프롬프트를 제공하는 경우, placeholder 안에 `<textarea>` 로 삽입.
클릭 한 번으로 전체 선택되고, Ctrl+A 전체 복사 시 textarea 내용은 자동 제외됨.

```html

  <span>🖼</span>
  <div class="label">이미지 N 삽입 — 설명</div>
  <div class="prompt-label">▼ ChatGPT 이미지 프롬프트 (클릭 후 Ctrl+A → Ctrl+C)</div>
  <textarea class="prompt-box" readonly rows="10" onclick="this.select()">
[프롬프트 내용]
  </textarea>
</div>
```

## 14. 자동화 포스팅 작성 가이드 (프롬프트 입력 규격)

자동화 시 아래 항목을 입력으로 받아 포스팅 생성:

```yaml
카테고리: [홈페이지|케이스스터디|동영상|정부지원사업|알지비뉴스]
제목: ""
포스팅_유형: [사례소개|서비스소개|정보전달|시리즈N편]
고객사명: ""   # 사례 포스팅 시
프로젝트_핵심: ""  # 1~2줄 요약
주요_서비스_키워드: []  # 해시태그에 활용
시리즈_정보:   # 시리즈 포스팅 시
  현재편: 1
  전체편수: 3
  다음편_주제: ""
```

### 자동화 작성 순서
1. 인삿말 (카테고리별 패턴 적용)
2. 포스팅 소개 (프로젝트/주제 1~2문장)
3. 페인포인트 또는 트렌드 (공감대 형성)
4. 본론 소제목 2~4개 + 각 설명
   - 핵심 키워드 굵게 + 반복 적용
   - 불릿 포인트 적절히 삽입
5. 마무리 + CTA (카테고리별 패턴 적용)
